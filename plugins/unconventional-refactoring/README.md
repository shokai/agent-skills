# unconventional-refactoring

定石外発想で実装をリファクタリングするAgent Skillです。実装済みの解の裏にある暗黙の前提条件を1つずつ外し、よりシンプルな別解がないかを検討します。

## 背景

AIに問題解決を頼むと、どうしても「現状の発展型」しか出てきません。Skillを改善させると記述を足し、プログラムを改善させると機能を足す方向に行きがちで、ソフトウェア開発の定石に素直に従いすぎて過剰な実装になります。

人間は並行して別のことをしています。既知の定石を連想しつつ、同時に「枷＝前提条件」を1つずつ外し、「この前提が無ければもっと簡単に解けるのではないか」という当たりを付けています。このスキルは、その人間がやっている手順をAgentに実行させます。

`批判的思考の連鎖` で「やり方や前提まで疑え」と指示しても足りません。問題解決において前提を疑う _方法_ をAgentは持っていないためです。このスキルはその方法を手順として与えます。

## このスキルがやること

実装済みの解を対象に、以下を行います。

1. 解の裏にある暗黙の前提条件を、コードと会話コンテキストから列挙する
2. 各前提を1つずつ外し、よりシンプルな別解がないか検討する
3. シンプルになった別解を5観点で比較する。実装量 / 既存仕様との互換性 / 説明コスト / 運用・移行コスト / 将来の保守性
4. 前提解除を成立させるために必要な別の仕組みのコストも計上する
5. 「今すぐ検討すべき / 将来の論点 / 現状の採用案で妥当」に3分類して報告する

提案・分類で停止します。コードの自動適用はしません。どの別解を採用するかは人間が判断します。

## 使いどころ

既にある実装や直前にまとまった設計案を、もっと少ない実装・説明で済ませられないか見直すときに使います。

ゼロから新しい案を作るためではなく、採用済みの解の前提を外して再考するためのスキルです。

## 前提条件

- `git` が利用できること。直近の変更を分析対象として確認するために使用します
- `codex-consultation` スキル: 任意。ユーザーが相談を求めた場合に、外部Agentへ独立して前提解除させるために使用します
  - 未インストールの場合は `subagent-consultation` にフォールバックします
- `subagent-consultation` スキル: codex-consultationが利用できない場合のフォールバック先
  - 両方未インストールでもAgent単独で動作します

## 使い方

```bash
/unconventional-refactoring [対象ファイル/モジュール]
```

作業中に「定石外発想で」「前提を疑って」「もっとシンプルにできないか」と伝えると発動します。引数を省略した場合は、会話コンテキストまたは直近の変更から対象を特定します。
